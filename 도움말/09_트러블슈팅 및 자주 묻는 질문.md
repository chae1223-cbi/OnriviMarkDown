# 09_트러블슈팅 및 자주 묻는 질문 (FAQ)

온리비 어서(Onrivi Author)를 사용하며 발생할 수 있는 주요 이상 현상에 대한 진단과 해결책, 그리고 안정적인 글쓰기를 돕기 위한 팁 모음입니다.

---

## 🙋‍♂️ Q1. 다른 창으로 넘어가거나 마우스를 밖으로 빼면 마지막 한 글자가 사라지거나 두 번 적혀요!
* **현상**: 한글 문장을 한참 작성 중인 상태(초성/중성/종성이 결합 중인 상태)에서 브라우저 바깥 영역을 클릭하거나 웹 서칭을 위해 크롬창으로 이동(Focus Out)하면, 미처 완료되지 않은 마지막 글자가 지워지거나 반대로 두 번(예: `글자` -> `글자자`) 적히는 고질적인 한글 입력기(IME) 버그입니다.
* **해결책**: 온리비 어서에는 **포커스 유실 실시간 구출 알고리즘**이 적용되어 있습니다. 사용자가 창을 벗어나는 즉시 `onDidBlurEditorText` 리스너가 가동되어 에디터 버퍼 내부의 미완성 원문을 React 본문 데이터로 강제 결합(Flush)합니다. 사용자는 버그 발생에 대한 우려 없이 자유롭게 화면과 다른 프로그램을 오가며 필기를 이어가실 수 있습니다.

---

## 🙋‍♂️ Q2. 추천 리스트가 떴을 때 단어 선택 후 Tab을 누르면 엉뚱하게 공백이 생깁니다!
* **현상**: 단어 자동완성 팝업(Suggest Widget)이 띄워진 상태에서 목록 중 원하는 추천 단어를 고르고 `Tab` 키를 눌렀을 때, 단어가 완성되는 대신 단순 탭 여백(`  `)이 들어가는 경우입니다.
* **해결책**: 온리비 어서의 최신 업데이트를 통해 탭(Tab) 키 조작 규칙이 개선되었습니다. 
  1. 자동완성 추천창이 열려 있을 때는 `Tab` 키가 **단어 완성 수락**으로 최우선 매핑됩니다.
  2. 마크다운 리스트 목록 줄에서 `Tab` 키를 누르면 **목록 들여쓰기**로 동작합니다.
  3. 그 외 일반 입력 화면에서는 기본 **공백 추가** 기능이 수행됩니다.
  자동완성 팝업이 활성화되어 있다면 단어를 고른 후 안심하고 `Tab`을 누르시면 됩니다.

## 🙋‍♂️ Q3. 상단 메뉴바를 클릭해도 하위 메뉴 팝업이 전개되지 않거나 가려서 안 보여요!
* **현상**: 상단의 파일, 편집, 보기 등 대메뉴를 클릭했으나 하위 드롭다운 메뉴가 아래 레이아웃에 파묻히거나 잘려서 클릭할 수 없던 현상입니다.
* **원인 및 조치**: 화면 스크롤 반응형을 유지하는 부모 레이아웃의 `overflow-y-hidden` 속성이 자식 드롭다운 메뉴 창까지 강제로 숨겨 버리는 화면 구조 버그였습니다. 최신 패치에서 메뉴바 컨테이너 구조를 정돈하여 레이아웃의 독립성을 확보했으며, 클릭 시 드롭다운 팝업 메뉴가 깔끔하게 가장 앞 레이어로 펼쳐집니다.

## 🙋‍♂️ Q4. 로컬 이미지를 불러왔는데 미리보기 창에 엑박(깨진 이미지)으로 뜹니다!
* **원인**: 최신 웹 브라우저나 데스크톱 Electron 보안 규칙은 보안 예방조치로 물리 하드디스크 파일 경로(`file:///C:/...`)에 임의 접근하는 행위를 엄격하게 차단합니다. 
* **해결책**: 온리비 어서 내부에 탑재된 미디어 프록시 핸들러가 탑재되었습니다. 사용자가 임의의 드라이브 이미지를 호출할 경우, `fs.createReadStream` 스트리밍 파일 파이프라인을 거쳐 적합한 MIME Type 및 CORS 헤더를 강제 주입하여 안전한 로컬 네트워크 자원 형태로 미리보기 화면에 뿌려줍니다. 이미지의 주소 경로가 컴퓨터에 실재하는 경로가 맞는지 확인해 주세요.

## 🙋‍♂️ Q5. 본문 작성 중에 깔끔하게 아래 각주 설명란으로 커서를 한 번에 보낼 수 없나요?
* **해결책**: 상단 툴바의 **📌(각주) 단추**를 누르면, 작성 중인 위치에 각주 링크 번호가 부여됨과 동시에 화면 하단 끝에 각주 상세 작성용 포인터(`[^번호]: `)가 자동 세팅됩니다. 
* 더불어, 사용자가 아래로 화면을 내려 수동으로 커서를 클릭하지 않아도 에디터 커서 포커스가 **자동으로 최하단 각주 서술 행으로 강제 이동**합니다. 해당 각주 내용 서술을 마친 후 다시 본문 영역을 클릭하여 필기를 원활하게 재개해 보세요.

## 🙋‍♂️ Q6. 애드온(웹) 버전에서 새로 실행할 때마다 '파일을 보고 복사하도록 허용하시겠습니까?' 팝업이 뜨는데 안 뜨게 할 수 없나요?
* **원인**: 구글 크롬 브라우저의 기본 보안 정책인 **File System Access API 보안 규정** 때문입니다. 웹 사이트나 크롬 확장 프로그램이 사용자 PC의 로컬 폴더에 직접 접근하는 권한은 세션(브라우저가 켜져 있는 동안) 단위로만 유지되며, 브라우저 재실행 또는 페이지 새로고침 시 즉시 만료되어 사용자 동의 팝업이 다시 노출됩니다.
* **해결책**: 이 팝업을 완전히 피하고 싶으시다면 브라우저 기반의 애드온 대신 **온리비 어서 데스크톱 앱(Electron) 버전**을 사용하시는 것을 적극 권장합니다. 데스크톱 앱은 네이티브 파일 권한을 직접 활용하므로 승인 팝업이 일절 발생하지 않습니다.
