# 06_마크다운 문법 입문 가이드

온리비 어서(Onrivi Author)에서 글을 쓸 때 활용할 수 있는 표준 마크다운(Markdown) 문법 가이드입니다. 
마크다운을 사용하면 마우스로 텍스트를 드래그하거나 글꼴 메뉴를 뒤적이지 않고도 키보드 타이핑만으로 직관적으로 문서의 서식을 완성할 수 있습니다.

---

## 1. 제목 (Headers)
문서의 제목이나 문단 구조를 설계할 때 `#` 기호를 사용합니다. `#`의 개수가 많아질수록 폰트 크기가 작아집니다.
* `#` 뒤에는 반드시 **한 칸의 공백(Space)**을 띄워야 정상 작동합니다.

```markdown
# 제목 1단계 (H1) - 책/문서의 제목
## 제목 2단계 (H2) - 대분류 / 장(Chapter)
### 제목 3단계 (H3) - 중분류 / 절(Section)
#### 제목 4단계 (H4) - 소분류
##### 제목 5단계 (H5) - 세부 단락
###### 제목 6단계 (H6) - 최소 단위 제목
```

---

## 2. 텍스트 강조 서식 (Text Formatting)
단어나 문장을 돋보이게 하기 위한 서식입니다. 온리비 어서 에디터에서는 텍스트를 마우스로 드래그한 상태에서 상단 툴바의 서식 단추나 단축키를 누르면 자동으로 씌워집니다.

| 서식 종류 | 마크다운 문법 | 출력 결과 | 단축키 |
| :--- | :--- | :--- | :--- |
| **굵게 (Bold)** | `**텍스트**` 또는 `__텍스트__` | **텍스트** | `Ctrl + B` |
| *기울임 (Italic)* | `*텍스트*` 또는 `_텍스트_` | *텍스트* | `Ctrl + I` |
| ~~취소선 (Strike)~~ | `~~텍스트~~` | ~~텍스트~~ | (툴바 버튼) |
| `인라인 코드` | `` `텍스트` `` | `텍스트` | (툴바 버튼) |

---

## 3. 목록 및 리스트 (Lists)
생각을 일목요연하게 정리하거나 목록을 나타낼 때 사용합니다. 
* 목록 작성 중 **`Tab`**을 누르면 계층을 한 단계 안으로 들여쓰고, **`Shift + Tab`**을 누르면 한 단계 밖으로 내어씁니다.

### ① 순서 없는 글머리 기호 목록
`-`, `*`, `+` 기호 중 하나를 입력한 후 한 칸 띄우고 내용을 입력합니다.
```markdown
- 첫 번째 항목
  - 한 단계 들여쓴 세부 항목 (Tab 입력)
- 두 번째 항목
```

### ② 순서 있는 번호 목록
숫자와 마침표(`.`)를 입력한 후 한 칸 띄우고 내용을 입력합니다.
```markdown
1. 첫 번째 순서
2. 두 번째 순서
3. 세 번째 순서
```

### ③ 체크리스트 (태스크 목록)
일정 관리나 작성 현황을 체크할 수 있는 리스트입니다.
```markdown
- [ ] 아직 완료하지 않은 항목
- [x] 완료한 항목 (대괄호 안에 영문 x 표시)
```

## 4. 인용구 및 구분선 (Blockquote & Divider)

### ① 인용구 (Blockquote)
명언이나 외부 글을 인용할 때 문단 맨 앞에 `>` 기호를 붙입니다.
```markdown
> "인생은 웅장한 여정이며, 글쓰기는 그 여정을 남기는 가장 아름다운 방법이다."
```

### ② 구분선 (Horizontal Rule)
내용의 단락을 나눌 때 `---` 또는 `***`을 입력하고 엔터를 칩니다.
```markdown
단락이 끝난 지점입니다.
---
새로운 단락이 시작되는 지점입니다.
```

---

## 5. 표 (Tables) 및 표 내부 줄바꿈
수직선(`|`)과 하이픈(`-`)을 조합하여 깔끔한 표를 그립니다.

### ① 기본적인 표 작성법
```markdown
| 도서명 | 저자 | 출판일 |
| :--- | :---: | ---: |
| 온리비 저작도구 | 채병익 | 2026년 |
| 마크다운 가이드 | 홍길동 | 2025년 |
```
* **정렬 조건**: 
  - `:---` : 왼쪽 정렬 (기본값)
  - `:---:` : 가운데 정렬
  - `---:` : 오른쪽 정렬

### ② 💡 표 셀(Cell) 안에서 줄바꿈하는 법
마크다운 표 내부에서 단순히 `Enter` 키를 치면 표 형식이 완전히 깨집니다. 
셀 내부에서 다음 줄로 행을 바꾸고 싶다면 **`Shift + Enter`**를 누르세요. 표 내부에서 줄바꿈이 정상적으로 구현되는 `<br>` 태그가 커서 자리에 자동으로 삽입됩니다.

## 6. 코드 블록 (Code Block)
소스 코드나 텍스트를 가공하지 않고 원본 그대로 강조해서 표시해야 할 때 사용합니다. 키보드 숫자 1번 왼쪽에 있는 백틱 기호(```) 3개를 감싸서 표현합니다.
* 여는 백틱(```) 오른쪽에 `javascript`, `python`, `html` 등의 언어 명칭을 명시하면 해당 프로그래밍 언어의 색상 강조가 작동합니다.

\`\`\`javascript
function welcome() {
    console.log("온리비 어서에 오신 것을 환영합니다!");
}
welcome();
\`\`\`

## 7. 📌 지능형 각주(Footnote) 활용하기
문서 본문 중 부연 설명이 필요한 특정 단어 옆에 주석 번호를 매기고, 문서의 맨 끝에 세부 설명을 적는 방식입니다.

1. 본문 중 각주를 넣을 위치에서 툴바의 **📌(각주) 버튼**을 누릅니다.
2. 커서 자리에 `[^1]` 형태의 링크가 생성되며, **문서의 맨 밑바닥에 `[^1]: ` 공간이 자동으로 생성**됩니다.
3. 편집 포커스가 자동으로 문서 최하단 각주 영역으로 가므로, 그 상태에서 상세 설명을 타이핑하면 끝납니다. 

```markdown
마크다운은 간단한 텍스트 기호[^1]를 활용하여 문서를 서식화하는 문법입니다.

... (중략) ...

[^1]: 텍스트 기반 마크업 언어의 일종으로 2004년 존 그루버에 의해 개발되었습니다.
```
