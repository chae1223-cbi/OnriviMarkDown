# 제품 요구사항 정의서 (PRD): Onrivi Docu-Agent

## 1. 프로젝트 개요 (Overview)
*   **프로젝트명:** 문서 기반 프로젝트 생애주기 관리(ALM) 및 자동화 플랫폼 (가칭: Onrivi Docu-Agent)
*   **컨텍스트:** 기존 Onrivi Author의 에디터 기능을 기반으로 확장된 AI 자율형 ALM 플랫폼.
*   **개념:** 아이디어 빌딩부터 구현, 테스트, 배포에 이르는 소프트웨어 개발 전 과정의 문서를 유기적으로 자동 생성하고 동기화하는 AI 에이전트 시스템.

## 2. 비전 및 핵심 가치 (Vision & Value Proposition)
1.  **코딩 환경의 자유:** 개발자의 익숙한 IDE 환경을 존중하며, 문서와 맥락의 컨트롤 타워 역할에 집중.
2.  **단일 신뢰 소스 (SSoT):** 요구사항 수정 시 설계, 테스트, 배포 스크립트가 실시간으로 상호 동기화.
3.  **의미론적 연결 (Semantic Link):** 기능 단위를 기준으로 UI-API-DB-테스트-배포 단계를 구조화된 데이터로 연결.

## 3. 핵심 기능 요구사항 (Product Features)

### 3.1. [설계] 아이디어 기반 요구사항 및 아키텍처 자동화
| ID | 기능명 | 상세 요구사항 | 우선순위 |
| :--- | :--- | :--- | :--- |
| **REQ-S-01** | 한 줄 요구사항 해석기 | 자연어 아이디어를 분석하여 표준 기능 정의서 및 PRD 초안 자동 생성 | High |
| **REQ-S-02** | 의미론적 설계 매핑 | 기능 기준 UI 컴포넌트, API 구조, DB 테이블 정의서 유기적 파생 | High |
| **REQ-S-03** | 표준 포맷 내보내기 | Swagger(OAS), Markdown, JSON 규격 변환 및 다운로드 지원 | Medium |

### 3.2. [테스트] 설계 문서 기반 검증 시나리오 자동화
| ID | 기능명 | 상세 요구사항 | 우선순위 |
| :--- | :--- | :--- | :--- |
| **REQ-T-01** | 인수 테스트 시나리오 생성 | 비즈니스 로직 기반 QA 체크리스트 테이블 자동 생성 | High |
| **REQ-T-02** | API/DB 테스트 스니펫 | Curl 및 Postman Mock 데이터를 문서 내에 내장 제공 | Medium |
| **REQ-T-03** | 역동기화 (Reverse Sync) | 테스트 중 변경 사항을 설계 문서에 실시간 역반영 | Low |

### 3.3. [구축] 인프라 명세 및 배포 가이드 자동화
| ID | 기능명 | 상세 요구사항 | 우선순위 |
| :--- | :--- | :--- | :--- |

| **REQ-D-02** | IaC 및 CI/CD 스크립트 | Docker, GitHub Actions, Terraform 코드 블록 제공 | Medium |
| **REQ-D-03** | 자동 릴리즈 노트 생성 | 버전 비교를 통한 변경 기능 점검 및 고객 공유용 문서 추출 | Medium |

## 4. 기술 스택 및 전략 (Technical Strategy)
*   **플랫폼:** Next.js (App Router), TypeScript, Vercel
*   **AI 엔진:** Gemini 1.5 Flash API (고성능 경량 처리)
*   **데이터 관리:** 문서 기반 구조화 데이터 (Markdown + JSON)
*   **MVP 아웃라인:** [아이디어] -> [PRD/API/DB 설계] -> [테스트/배포 가이드] 통합 패키지 출력
