# 구현계획서: 미리보기 리스트 번호 및 빈 줄 렌더링 수정

## 1. 배경 및 문제 정의

### 1.1. 번호 강제 상속 (Numbering Inheritance)

마크다운에서 다음 패턴으로 작성 시:

```markdown
1. 항목 A
2. 항목 B

본문 단락

1. 항목 C
```

우측 미리보기는 `1. 항목 C`를 `3. 항목 C`로 표시한다. 원인은 이중 구조:

1. **CommonMark 파싱**: 빈 줄 + 본문 단락이 있으면 리스트가 분할되어 두 번째 `<ol>`가 생성되지만, `start` 속성이 remark AST에서 소실되어 브라우저 기본값인 `1`부터 시작
2. **HTML `<ol>` 기본 동작**: `<li>` 순서대로 브라우저가 자동 번호 부여 → 첫 번째 리스트가 2개였다면 두 번째 리스트의 첫 `<li>`는 `3`으로 표시

### 1.2. 빈 줄 압착 소멸 (Blank Line Collapsing)

```markdown
문장 끝.

(여기 빈 줄 1개)

(여기 빈 줄 1개)

새 문단
```

미리보기에서 빈 줄이 간격 없이 사라짐. 원인: HTML의 **whitespace collapsing** 규칙이 빈 `p`, `br` 요소의 물리 높이를 0으로 압착.

### 1.3. 중첩 리스트 여백 2배 증폭 (Nested List Double Padding)

```
1. 항목
   - 서브 항목      ← 탭 1번
     - 서브서브     ← 탭 2번 (여백 과다)
```

단계가 깊어질수록 브라우저 네이티브 `padding-left` + Tailwind Typography 기본 마진이 중첩 누적되어 광활하게 벌어짐.

## 2. 변경 대상 파일 및 분석

| # | 파일 | 현재 상태 | 변경 필요 |
|---|------|-----------|-----------|
| ① | `MarkdownViewer.tsx` | `ol` override에 `start` 미전달, `br` override 없음 | `start` 속성 전달 + 빈 줄 스타일링 |
| ② | `globals.css` | `.prose ul, ol` 기본 마진만 존재, nested 리스트 규칙 없음, `br` 스타일링 없음 | nested 리스트 패딩 교정, 빈 줄 높이 고정 |
| ③ | `preprocessMarkdown` (확인 필요) | 전처리기가 빈 줄이나 리스트 번호를 변경하는지 | 수정 불필요로 예상 |

### 2.1. 선행 확인 사항

- `remarkBreaks`(line 4)와 `remarkGfm`(line 3)은 **이미 MarkdownViewer.tsx에 import 및 사용 중** → 제아이 제안 중복
- `<ol start={...}>` 속성은 react-markdown이 remark AST에서 자동 전달하지만, 현재 override에서 `{...props}`로 전달 중 → **확인 및 테스트 필요**
- CSS 클래스는 `.markdown-preview`가 아닌 **`.prose`** (Tailwind Typography) 사용 중

## 3. 상세 변경 명세

### 3.① `MarkdownViewer.tsx` — `ol` override에 `start` 속성 명시적 전달

**현재 (838행):**
```tsx
ol: ({ node, children, style, ...props }) => <ol style={style} {...props}>{children}</ol>,
```

**변경:**
```tsx
ol: ({ node, children, style, start, ...props }) => (
  <ol start={start} style={style} {...props}>{children}</ol>
),
```

#### 3.①-1. 동작 설명

- `react-markdown` v9+는 remark AST의 `start` 속성을 컴포넌트 prop으로 전달
- `start`가 `undefined`(첫 번째 리스트나 시작 번호 `1`)면 HTML `start` 속성이 생략되어 브라우저 기본값 `1` 사용
- `start`가 `4`면 `<ol start="4">`가 되어 브라우저가 `4`부터 카운팅
- 단, CommonMark는 빈 줄 1개만으로는 리스트를 분할하지 않음 → 사용자가 **본문 단락(paragraph)** 을 넣어야 리스트가 나뉘고 `start`가 적용됨

#### 3.①-2. 사용자 교육 필요사항

리스트 분할을 의도할 때는 반드시 아래 패턴을 따라야 함:
```markdown
1. 첫 번째 리스트
2. 계속

(본문 설명)

1. 새 리스트 시작
```
빈 줄만으로는 리스트가 분할되지 않으므로, 간격만 원한다면 CSS로 해결(3.② 참조).

### 3.② `globals.css` — 리스트 및 빈 줄 CSS 보강

#### 3.②-1. 빈 줄 물리 높이 보장 (제아이 제안 보정)

현재 `remarkBreaks`가 `\n`을 `<br>`로 변환하고, `rehypeBrRaw`가 raw `<br>`을 처리한다. 브라우저가 `<br>`을 0 높이로 압착하지 않도록 CSS 추가:

```css
/* 미리보기 영역 내 br 요소에 물리 높이 부여 */
.prose br {
  display: block !important;
  content: "" !important;
  margin-top: 1.2rem !important;
  height: 0 !important;
}

/* 빈 p 태그 높이 붕괴 방어 */
.prose p:empty::before {
  content: "\00a0" !important;
  display: block !important;
}
```

**설계 근거:**
- `display: block`은 `br`이 인라인에서 블록으로 바뀌어 높이를 가짐
- `margin-top: 1.2rem`은 `line-height: 1.8`(기본 프로필) 대비 빈 줄 1개 분량
- `height: 0`으로 `margin`만으로 높이를 표현하여 중복 높이 방지
- `p:empty::before`는 빈 `<p>`가 완전히 소멸되는 것을 방어

**주의:** `remarkBreaks`가 생성한 `<br>`과 유저가 직접 입력한 `<br>`이 모두 영향받음. 현재 `rehypeBrRaw` 플러그인이 raw `<br>`도 처리하므로 일관된 동작 예상.

#### 3.②-2. 중첩 리스트 패딩 교정

**현재 (192~200행):**
```css
.prose ul, .prose ol {
  margin-top: 0.5em !important;
  margin-bottom: 0.5em !important;
}
.prose li {
  margin-top: 0.2em !important;
  margin-bottom: 0.2em !important;
}
```

**변경:**
```css
.prose ul, .prose ol {
  margin-top: 0.5em !important;
  margin-bottom: 0.5em !important;
}

/* 🚨 [중첩 리스트 여백 폭발 진압] 부모 여백이 중첩되어 2배로 먹히는 현상 차단 */
.prose ol ol, .prose ol ul, .prose ul ol, .prose ul ul,
.prose li ol, .prose li ul {
  margin-top: 0 !important;
  margin-bottom: 0 !important;
  padding-left: 1.5rem !important;
}

.prose li {
  margin-top: 0.2em !important;
  margin-bottom: 0.2em !important;
}
```

**설계 근거:**
- 브라우저는 `<li>` 내부 `<ol>/<ul>`에 기본 `padding-left: 40px` 적용
- Tailwind Typography(prose)가 여기에 추가 마진 부여 → 중첩될수록 2배 누적
- `margin-top/bottom: 0`으로 부모 마진의 재적용 차단
- `padding-left: 1.5rem`으로 단일 고정폭 사용 (단계별 증가 없음)

#### 3.②-3. 타깃 셀렉터 정밀화

제아이의 제안은 `.markdown-preview` 클래스를 사용했으나, 실제 DOM 구조는:

```html
<div class="prose prose-sm md:prose-base dark:prose-invert max-w-none custom-preview-container">
  <div class="max-w-4xl mx-auto w-full">
    <div class="markdown-body">
      <!-- react-markdown 렌더링 결과 -->
    </div>
  </div>
</div>
```

`.prose`가 최상위 컨테이너이므로 모든 CSS는 `.prose` 셀렉터로 적용. `custom-preview-container`는 동적 CSS 프로필에만 사용되므로 무관.

### 3.③ `preprocessMarkdownForPreview` 확인

전처리기가 리스트 번호나 빈 줄을 의도치 않게 변경하는지 검증 필요.

**확인 위치:** `frontend/src/lib/markdownPreprocessor.ts` 또는 유사 파일

**검증 항목:**
- `\n\n`(빈 줄)을 `\n`으로 압축하는 로직이 있는가
- 번호 리스트 마커(`1.`, `2.`)를 재작성하는 로직이 있는가
- 들여쓰기 탭을 공백으로 변환하면서 리스트 구조가 깨지는가

→ 위 항목이 있다면 해당 로직 제거 또는 보정 필요

## 4. 테스트 계획

### 4.1. 리스트 번호 분할

| 입력 | 예상 결과 |
|------|-----------|
| `1. A\n2. B\n\n본문\n\n1. C` | 두 번째 리스트가 `1. C`로 표시 |
| `1. A\n2. B\n\n1. C` | CommonMark 동일 리스트 → `3. C`로 표시 (정상 동작) |
| `1. A\n   1. B\n   2. C\n2. D` | 중첩 리스트가 합리적 간격으로 표시 |

### 4.2. 빈 줄

| 입력 | 예상 결과 |
|------|-----------|
| `A\n\n\nB` | A와 B 사이에 빈 줄 2개 분량 간격 |
| `A\n<br>\n<br>\nB` | 동일한 간격 |

### 4.3. 중첩 리스트 여백

| 입력 | 예상 결과 |
|------|-----------|
| `- A\n  - B\n    - C` | 단계당 `1.5rem` 패딩, 2배 증폭 없음 |
| `1. A\n   1. B\n      1. C` | 동일 |

## 6. 위험 요소

| 위험 | 영향 | 대응 |
|------|------|------|
| `remarkBreaks`의 `<br>` 변환이 빈 줄 수와 무관하게 모든 `\n`에 `<br>` 생성 | 빈 줄 의도와 다른 렌더링 | `globals.css`에서 `br` 높이만 조정. 근본적 해결은 `preprocessMarkdownForPreview`에서 `\n\n` 패턴 유지 |
| `start` prop이 react-markdown 버전에 따라 전달되지 않음 | 번호 분할 무효 | 업그레이드 시 테스트 필요. 현재 `"react-markdown": "^9.0.0"` 사용 확인 |
| CSS `!important` 과다 사용으로 사용자 CSS 프로필 충돌 | 프로필 커스터마이징 무력화 | `prose` 기본 셀렉터에만 적용하고 프로필 CSS(`custom-preview-container`)보다 낮은 특이도 유지 |

## 7. 제아이 분석 대비 보정 사항

| 제아이 제안 | 실제 코드 반영 | 이유 |
|-------------|---------------|------|
| `remarkBreaks` 플러그인 추가 | **불필요** (이미 존재) | MarkdownViewer.tsx:4에 import, 666행에서 사용 중 |
| `.markdown-preview` 클래스 | **`.prose`로 대체** | 실제 DOM 클래스는 Tailwind Typography의 `prose` |
| `<br>` 컴포넌트 override | **불필요** | `remarkBreaks` + `rehypeBrRaw`가 이미 처리. CSS로 높이 제어 |
| `ol`에 `start` 전달 | **필요** | 현재 `{...props}`로 전달 중이지만 명시적 확인 필요 |
| `ol ol` 중첩 패딩 CSS | **필요** | 현재 globals.css에 없음 |
| 빈 줄 `br` CSS | **필요** | 현재 globals.css에 없음 |
