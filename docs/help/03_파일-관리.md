온리비 어서는 세 가지 워크스페이스 모드를 지원합니다.

## 워크스페이스 모드

### 로컬 파일 시스템 (Electron)

Electron 데스크톱 앱에서는 운영체제의 파일 시스템에 직접 접근하여 문서를 저장하고 관리합니다.

- **저장**: `Ctrl+S` — 현재 문서를 저장
- **다른 이름으로 저장**: `Ctrl+Alt+S` — 새 파일명/위치로 저장
- **폴더 열기**: 메뉴 > 파일 > 폴더 열기 — 폴더 단위로 문서를 탐색하고 관리

### 브라우저 File System Access API (로컬 웹)

최신 브라우저(Chrome, Edge)에서는 File System Access API를 통해 로컬 폴더에 접근할 수 있습니다.

- **폴더 선택**: "폴더 열기" 버튼을 클릭하여 브라우저의 폴더 선택 다이얼로그를 통해 작업할 폴더를 지정
- **파일 읽기/쓰기**: 선택된 폴더 내에서 파일을 읽고 저장
- **권한**: 브라우저가 폴더 접근 권한을 관리하며, 최초 접근 시 사용자 승인이 필요

### 가상 파일 시스템 (VFS / Chrome 확장 프로그램)

Chrome 확장 프로그램 환경에서는 브라우저의 IndexedDB를 기반으로 하는 가상 파일 시스템(VFS)을 사용합니다.

- **VFS 특징**:
  - IndexedDB에 파일 내용을 저장
  - 확장 프로그램 내에서 완전히 동작 (로컬 파일 시스템 접근 불필요)
  - 파일 트리, 폴더 생성/삭제, 파일 이동 등 기본적인 파일 관리 기능 지원
- **제한사항**: 실제 로컬 파일이 아닌 브라우저 내 저장소에 저장되므로, 다른 앱에서 직접 접근할 수 없습니다

## 파일 트리

좌측 사이드바의 **파일 탐색기** 탭에서 현재 워크스페이스의 파일과 폴더를 트리 형태로 확인할 수 있습니다.

- **파일 열기**: 파일 이름을 클릭
- **파일 생성**: 폴더 선택 후 `Ctrl+N` 또는 우클릭 메뉴
- **파일 삭제**: 파일 우클릭 > 삭제 (휴지통 아이콘)
- **파일 이름 변경**: 파일 우클릭 > 이름 변경
- **폴더 생성**: 우클릭 > 새 폴더

## 자동 저장

설정에서 자동 저장 간격을 지정할 수 있습니다. 자동 저장이 활성화되면 문서가 일정 시간마다 자동으로 저장됩니다.

- **기본값**: 30초
- **설정 위치**: 설정 > 에디터 > 자동 저장 간격
