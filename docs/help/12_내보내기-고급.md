온리비 어서의 내보내기 시스템은 CSS 프로필과 결합하여 전문가 수준의 출력 품질을 제공합니다. 

## 렌더링 엔진 이해하기

### html2canvas (PDF, PNG)

PDF와 PNG 내보내기는 **html2canvas** 라이브러리를 사용하여 DOM을 캡처합니다.

#### 알려진 한계와 대응

| 한계                          | 대응 방법                                               |
| ----------------------------- | ------------------------------------------------------- |
| `::before` / `::after` 미지원 | 목록 마커를 DOM 요소로 직접 주입 (`export-list-marker`) |
| 인라인 코드 높이 오계산       | display:inline-block + height/line-height 고정          |
| 한글 폰트 누락                | Nanum Gothic, Malgun Gothic 등 한글 폰트 폴백 적용      |
| 상대 경로 이미지 누락         | Base64 Data URI로 인라인 변환                           |

### 이미지 인라인 변환 프로세스

내보내기 시 상대 경로 이미지는 자동으로 Base64 Data URI로 변환됩니다:

1. **Electron**: IPC를 통해 로컬 파일을 읽어 Base64로 변환
2. **브라우저**: 백엔드 API(`/api/view`)를 통해 이미지 서빙 후 변환
3. **Web/HTTPS URL**: 그대로 유지 (변환 불필요)

### 목록 마커 처리

html2canvas는 CSS `::before`와 `counter()`를 지원하지 않아 목록 마커가 누락됩니다. 온리비 어서는 이를 해결하기 위해:
<!-- [page-break] -->
1. 각 `<li>` 요소에 `export-list-marker` span을 DOM으로 직접 추가
2. 순서 목록은 시작 번호부터 순차적으로 번호를 계산
3. 비순서 목록은 `•` (U+2022) 문자 사용

## CSS 프로필과 내보내기 통합

### 적용 순서

1. CSS 프로필의 `pageStyle.backgroundColor`가 PDF 페이지 배경색으로 사용됩니다
2. CSS 프로필의 여백값(margin)이 PDF 페이지 여백으로 사용됩니다
3. CSS 프로필의 `rules`가 DOM 복제본에 주입되어 캡처됩니다
4. `dynamicCssString` (CSS 프로필에서 생성된 동적 CSS)이 복제본에 추가됩니다

### 페이지 나누기 제어

CSS 프로필에서 제목 요소(h1~h6)별로 페이지 나누기 규칙을 설정할 수 있습니다:

| 설정               | 설명                                      |
| ------------------ | ----------------------------------------- |
| 페이지 나누기 전   | 요소 앞에서 페이지 분할                   |
| 페이지 나누기 후   | 요소 뒤에서 페이지 분할                   |
| 페이지 나누기 방지 | 요소가 페이지에 걸쳐 분할되지 않도록 방지 |

### 캡처 해상도

PDF와 PNG 내보내기는 `scale: 2`로 캡처하여 레티나 디스플레이에서도 선명한 출력을 보장합니다.

## PDF 맞춤 설정

### jsPDF 설정

PDF 생성에는 **jsPDF** 라이브러리가 사용됩니다:

- 페이지 크기: A4 (210mm × 297mm)
- 방향: 세로(Portrait) / 가로(Landscape) 선택 가능
- 단위: 밀리미터(mm)
- 이미지 포맷: JPEG (품질 98%)
<!-- [page-break] -->
### 페이지 분할 로직

콘텐츠가 한 페이지를 초과할 경우:

1. 페이지 나누기 마커(`page-break`)를 기준으로 1차 분할
2. 각 섹션의 높이가 A4 콘텐츠 영역을 초과하면 세로로 분할
3. 분할된 섹션을 순서대로 PDF 페이지에 배치

## 전자책(EPUB) 최적화

EPUB 내보내기 시 CSS 프로필의 스타일이 EPUB의 내부 스타일시트로 포함됩니다:

- EPUB 3.0 형식
- CSS 프로필의 `backgroundColor`가 EPUB 페이지 배경으로 설정
- CSS 프로필의 `fontFamily`가 EPUB 본문 폰트로 설정
- 이미지는 Base64 인라인 포함

## 팁과 모범 사례

1. **미리보기에서 먼저 확인**: 내보내기 전에 분할 모드에서 결과를 미리 확인하세요
2. **페이지 뷰 활성화**: 페이지 뷰를 켜면 PDF 출력 결과를 화면에서 미리 볼 수 있습니다
3. **CSS 프로필 여백**: PDF 여백은 CSS 프로필의 용지 여백 설정을 따르므로, 내보내기 전에 여백을 확인하세요
4. **이미지 경로 확인**: 상대 경로 이미지는 내보내기 전에 실제 파일이 존재하는지 확인하세요
5. **폰트 로딩**: 내보내기 전에 사용자 정의 폰트가 로드되었는지 확인합니다 (자동 대기)
