온리비 어서는 4가지 뷰 모드를 제공하여 작성과 검토를 유연하게 전환할 수 있습니다.

## 뷰 모드

### 편집 모드 (Edit)

에디터만 전체 화면에 표시됩니다. 마크다운 작성에 집중하고 싶을 때 사용합니다.

- 단축키: 우측하단의 모드전환툴바를 클릭한다.
- 메뉴바: 도구 < 화면보기 모드 < 편집모드
- 특징: 미리보기가 없으므로 성능이 가장 가볍습니다

### 분할 모드 (Both) — 기본값

좌측에 에디터, 우측에 미리보기가 동시에 표시됩니다.

- 단축키: 우측하단의 모드전환툴바를 클릭한다.
- 메뉴바: 도구 < 화면보기 모드 < 분할 모드
- **양방향 스크롤 동기화**: 에디터에서 스크롤하면 미리보기가 따라가고, 미리보기에서 스크롤하면 에디터가 따라옵니다
- 실시간으로 작성 결과를 바로 확인할 수 있어 가장 권장되는 모드입니다

### 미리보기 모드 (Preview)

미리보기만 전체 화면에 표시됩니다. 읽기 전용으로 문서를 검토할 때 사용합니다.

- 단축키: 우측하단의 모드전환툴바를 클릭한다.
- 메뉴바: 도구 < 화면보기 모드 < 미리보기 모드
- 완성된 문서를 출력 결과와 동일하게 확인할 수 있습니다
- 체크리스트를 실제로 클릭하여 상태를 변경할 수 있습니다

### 서식 정의 모드 (CSS Style)

CSS 프로필 편집기와 미리보기를 함께 표시합니다.

- 단축키: 우측하단의 모드전환툴바를 클릭한다.
- 메뉴바: 도구 < 화면보기 모드 < 서식 정의 모드
- 우측에서 CSS 프로필을 편집하고 좌측에서 실시간으로 결과를 확인합니다
- 서식 정의에 대한 자세한 내용은 [서식 정의 (CSS 프로필)](/docs/시작하기/서식-정의) 문서를 참고하세요

## 양방향 스크롤 동기화

분할 모드에서는 에디터와 미리보기의 스크롤이 동기화됩니다.

- **에디터 → 미리보기**: 에디터에서 스크롤하면 미리보기가 같은 위치로 이동
- 에디터에서 작업하거나 작업(행)위치를 클릭하면 미리보기도 동시에 화면이동및 화면에 내용이 적용됨

## 페이지 뷰 (Page View)

설정에서 페이지 뷰를 활성화하면 미리보기가 A4 용지 형태로 표시됩니다.

- 페이지 나누기 표시 (빨간 점선)
- PDF 내보내기 결과를 화면에서 미리 확인 가능
- 설정 위치: 설정 > 페이지 뷰 활성화
