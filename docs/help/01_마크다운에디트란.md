온리비 어서(Onrivi Author)의 심장인 마크다운 지식 저작 공간에 오신 것을 환영합니다!

**마크다운의 정의와 문법** 섹션에서는 텍스트 기반 경량 언어의 설계 철학을 이해하고, 온리비의 명품 렌더링 엔진과 2열 세로 툴바가 지원하는 표준 코어 문법을 나노 단위로 검증 및 마스터할 수 있도록 안내합니다.

## 1. 마크다운(Markdown)의 순수한 정의와 철학

**마크다운(Markdown)**은 2004년 존 그루버(John Gruber)와 아론 슈워츠(Aaron Swartz)가 공동 웹 저작을 위해 개발한 **텍스트 기반의 경량 마크업 언어(Lightweight Markup Language)**입니다.

### 💡 텍스트 자체의 주권 회복
기존의 HTML 문법은 글을 하나 쓰려면 `<h1>제목</h1>`, `<p>본문 <strong>강조</strong></p>` 처럼 가독성을 해치는 수많은 태그 배관들을 사람이 직접 타이핑해야 했습니다. 마크다운은 이러한 복잡한 태그 래퍼들을 완벽히 걷어내고, 인간이 직관적으로 이해할 수 있는 서식 기호들만 사용합니다.

### 💡 마크다운의 핵심 철학: 가독성(Readability)
존 그루버가 명시한 마크다운의 가장 중심적인 설계 목표는 **"읽기 쉬움"**입니다. 마크다운 문서의 원본 텍스트는 서식이 렌더링되지 않은 순수한 상태(Plain Text)로 보더라도, 그 자체로 일반 이메일이나 텍스트 문서처럼 유려하고 직관적으로 읽힐 수 있어야 합니다. 마우스 없이 오직 키보드만으로 문서의 아키텍처를 완벽히 통제할 수 있는 최고의 지식 저작 도구입니다.

## 2. 순수 마크다운 핵심 코어 문법 (Standard Syntax)

온리비 어서의 렌더링 엔진 및 전 세계 모든 마크다운 파서가 완벽하게 컴파일하는 표준 문법 리스트입니다.

### ① 제목 (Headings)
- 문서의 단락 계층 구조를 정의합니다. 문장 시작부에 우물 정 자(`#`)의 개수로 1단계부터 6단계까지 지정하며, 기호 뒤에는 반드시 **공백(Space) 한 칸**을 띄워야 무결하게 컴파일됩니다.
- 온리비 툴바 가동 시에는 **`HeadingSpinButton`**을 통해 숫자로 마우스 제어가 가능합니다.
  ```markdown
  # 제목 1단계 (HTML의 <h1>)
  ## 제목 2단계 (HTML의 <h2>)
  ### 제목 3단계 (HTML의 <h3>)
  #### 제목 4단계 (HTML의 <h4>)
  ##### 제목 5단계 (HTML의 <h5>)
  ###### 제목 6단계 (HTML의 <h6>)

```
### ② 인라인 텍스트 스타일 (Text Emphasis)
* 텍스트 내부의 특정 단어를 강조하거나 기울일 때 사용합니다. 온리비 아서는 툴바 단축키 `Ctrl + B` 및 `Ctrl + I`와 동기화됩니다.
```markdown
*기울임* 또는 _기울임_ (Italic)
**두껍게** 또는 __두껍게__ (Bold)
***기울이면서 두껍게*** (Bold & Italic)
~~취소선~~ (Strikethrough)
<u>밑줄</u> (Underline)
```

### ③ 목록 (Lists)

* 순서가 없는 목록과 순서가 있는 목록을 기하학적으로 정렬합니다. `Tab`과 `Shift + Tab`으로 들여쓰기 깊이를 통제합니다.
```markdown
# 순서 없는 목록 (글머리 기호)
* 별표, 대시(-), 플러스(+) 모두 사용 가능합니다.
  * 탭(Tab) 또는 공백 4칸을 주입하면 하위 하이퍼 링크 구조로 들여쓰기됩니다.
# 순서 있는 목록 (숫자)
1. 첫 번째 항목
2. 두 번째 항목
3. 세 번째 항목 (번호를 모두 '1.'로 적어도 파서 단에서 자동으로 시퀀스 정렬을 수행합니다.)
# 체크박스
-[ ] 남자
-[X] 여자
```

### ④ 인용구 (Blockquotes)

* 타인의 글이나 핵심적인 아키텍처 비전을 격리 배치하여 강조할 때 사용하며, HTML의 "<blockquote>" 태그로 변환됩니다. V2 프로필 적용 시 고급스러운 파스텔톤 박스로 렌더링됩니다.
```markdown
> "당신의 생각은 소중합니다. 우리는 그 생각을 가장 아름답고 머물 만한 공간으로 만듭니다."
> 이 구역은 온리비 엔진 내부에서 명품 레이아웃 박스로 컴파일됩니다.

```

### ⑤ 코드 영역 (Code Blocks & Inline Code)
* 개발자나 기술 기획자들이 소스코드를 원문 그대로 보존하기 위한 영역입니다. 온리비의 Monaco Core 환경에서 탭 간격 4칸(`tab-size: 4`)이 강제 적용됩니다.

```markdown
# 인라인 코드 (문장 중간에 삽입)
본문 단락 내부에서 변수를 지정할 때는 `const onrivi = true;` 처럼 백틱(`) 가드를 칩니다.

# 코드 블록 (독립된 소스코드 공간)
백틱 3개(```)를 위아래로 배치하여 대량의 코드를 격리합니다.

```

### ⑥ 링크 및 이미지 (Links and Images)

* 웹상의 주소나 그래픽 자산을 문서 내부에 유기적으로 도킹합니다. 온리비 아서 툴바의 하이퍼링크 단축키 `Ctrl + K`와 매핑됩니다.
```markdown
# 하이퍼링크 연결
[온리비 오피셜 사이트](https://onrivi.com)

# 이미지 인젝션 (링크 구조 앞에 느낌표'!'를 붙입니다.)
![온리비 아서 로고 비주얼](https://onrivi.com/logo.png)

```



### ⑦ 수평 구분선 (Horizontal Rules)

* 장이 바뀌거나 주제가 대전환될 때 화면의 경계를 명확히 분할하는 수평선(`hr`)을 만듭니다. V2 서식 사상에 맞춰 **중앙 정렬 40% 슬림 비율**로 컴파일됩니다.
```markdown
---
***
___
(별표, 대시, 언더바를 3개 이상 연속 기입하면 슬림한 가드 라인이 컴파일됩니다.)

```
