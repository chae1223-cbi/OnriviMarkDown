# [Skill] Onrivi Author 문서 통합 갱신 지침 (v1.0)

이 문서는 MiniMD 프로젝트의 소스 코드 변경 시 기술 문서들을 최신 상태로 유지하기 위한 AI 전용 스킬 지침서입니다.

## 1. 문서 갱신 트리거 (Trigger)
다음과 같은 상황에서 이 스킬을 실행합니다:
- 새로운 UI 컴포넌트가 추가되었을 때
- 백엔드 API 엔드포인트가 신설되거나 규격이 변경되었을 때
- 시스템의 아키텍처(데이터 흐름)가 변경되었을 때
- 사용자가 "문서 전체 갱신해줘"라고 요청했을 때

## 2. 문서별 체크리스트 (Checklist)

### 2.1 03_API_Spec.md (API 명세)
- [ ] `backend/index.js`의 모든 `app.get`, `app.post` 등을 전수 조사한다.
- [ ] 파라미터(Query, Body)와 응답 데이터 형식이 현재 코드와 일치하는지 확인한다.

### 2.2 05_Project_Structure.md (구조도)
- [ ] `scripts/sync_docs.js`를 실행하여 물리적 디렉토리 트리를 최신화한다.
- [ ] `src/components` 내의 모든 신규 파일을 목록에 추가하고 기능을 정의한다.

### 2.3 06_Screen_Specification.md (화면 정의)
- [ ] 새로운 모달이나 팝업이 생겼는지 확인하여 ID(SCR-XX)를 부여한다.
- [ ] 각 화면의 주요 인터랙션(드래그 앤 드롭 등)이 설명에 포함되었는지 확인한다.

### 2.4 07_Component_Functional_Spec.md (컴포넌트 상세)
- [ ] 각 컴포넌트의 Props 인터페이스가 변경되었는지 확인한다.
- [ ] 컴포넌트 내의 핵심 함수(예: handleDrop)의 역할이 명세에 반영되었는지 확인한다.

## 3. 실행 방법 (How to run)
1. `scripts/sync_docs.js`를 실행하여 구조를 잡는다.
2. 각 문서(`01`~`07`)를 순차적으로 열어 코드와 대조하며 `replace_file_content`로 갱신한다.
3. 갱신이 완료되면 갱신된 항목 요약을 사용자에게 보고한다.
