# [Rules] Onrivi Docu-Agent 워크벤치 운영 규칙 (v1.0)

이 문서는 AI 에이전트(Antigravity)가 이 워크벤치 내에서 업무를 수행할 때 반드시 준수해야 하는 행동 지침입니다.

## 1. 업무 집중 및 범위 (Focus & Scope)

- **전념의 원칙:** AI 에이전트는 오직 **현재 프로젝트(Onrivi Docu-Agent / OnriviMarkDown)**와 관련된 업무에만 집중한다.
- **외부 요청 거절:** 현재 프로젝트 범위를 벗어나는 다른 프로젝트나 외부 업무 요청이 들어올 경우, **"이쪽 일이 아닙니다"**라고 정중히 답변하고 해당 업무를 수행하지 않는다.
- **컨텍스트 유지:** 모든 대화와 작업의 우선순위는 이 워크벤치 내의 요구사항 달성에 둔다.

## 2. 개발 방법론: 폭포수(Waterfall) 프로세스

모든 기능 개발은 다음의 선형적 절차를 엄격히 준수한다:

1. **요구사항 분석 (Requirements):** 사용자의 의도를 분석하고 `PRD` 및 `RTM`에 반영한다.
2. **시스템 설계 (Design):** 분석된 내용을 바탕으로 상세 설계 문서(`SPEC`)를 작성하고 사용자의 승인을 받는다.
3. **구현 (Implementation):** 승인된 설계 문서를 바탕으로 소스 코드를 작성한다.
4. **테스트 및 검증 (Test):** 구현된 기능이 설계 및 요구사항과 일치하는지 검증한다.

## 3. 추적성 관리 (Traceability)

- 요구사항의 변경이 발생하면 반드시 **RTM(Requirements Traceability Matrix)**을 먼저 업데이트한 후 하위 공정(설계, 구현)을 진행한다.
- 모든 코드 변경은 근거 문서(PRD, SPEC)가 존재해야 한다.
