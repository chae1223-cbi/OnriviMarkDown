# [Guide] Onrivi Author: 코드 재사용 및 자산화 가이드 (v1.0)

본 문서는 **Onrivi Author** 프로젝트에서 구현된 고품질의 로직과 컴포넌트를 향후 다른 프로젝트에서 효율적으로 참조하고 재사용하기 위한 방법을 정의합니다.

---

## 1. 재사용 가능한 핵심 모듈 (Core Assets)

프로젝트 내에서 독립성이 높아 즉시 추출 가능한 모듈 목록입니다.

### 1.1 파일 시스템 유틸리티 (`frontend/src/lib/helper.tsx`)
*   **Natural Sort 로직**: 한글, 영문, 숫자가 섞인 파일명을 윈도우 탐색기 방식으로 정렬하는 알고리즘.
*   **파일 확장자 분석기**: 확장자에 따른 Lucide 아이콘 매핑 로직.
*   **재귀적 폴더 스캔**: 브라우저 File System Access API를 활용한 디렉토리 트리 생성 로직.

### 1.2 보안 경로 처리 (`backend/index.js`)
*   **Path Traversal 방지**: `path.resolve`와 `startsWith`를 이용해 워크스페이스 외부 접근을 차단하는 보안 헬퍼 함수 (`getSafePath`).

### 1.3 디자인 시스템 토큰 (`frontend/src/app/globals.css`)
*   **Glassmorphism 유틸리티**: `.glass` 클래스를 통해 프리미엄 투명 효과를 즉시 적용 가능.

---

## 2. 컴포넌트 재사용 전략

### 2.1 UI 컴포넌트 추출
*   **대상**: `MenuBar.tsx`, `StatusBar.tsx`, `SettingsModal.tsx`
*   **방법**: 컴포넌트 파일과 해당 컴포넌트가 사용하는 `lib` 내의 타입을 함께 복사합니다.
*   **의존성**: `lucide-react`, `tailwind-merge`, `clsx` 패키지가 설치된 환경이라면 레이아웃 깨짐 없이 즉시 작동합니다.

### 2.2 비즈니스 로직 분리
*   **팁**: UI와 로직이 강하게 결합된 경우, 핵심 핸들러(예: 파일 저장, 검색 로직)를 별도의 Custom Hook(`useFileSystem.ts` 등)으로 분리하여 관리하면 재사용성이 더욱 높아집니다.

---

## 3. 아키텍처 재사용 (Patterns)

### 3.1 하이브리드 저장소 패턴
*   **개념**: 로컬 모드(REST API)와 브라우저 모드(IndexedDB)를 하나의 인터페이스로 추상화하여 제공하는 방식.
*   **적용**: 클라우드 저장소가 없는 소규모 웹 앱에서 데이터 영속성을 확보할 때 최적의 솔루션입니다.

### 3.2 전역 검색 아키텍처
*   **개념**: DB 없이 파일 시스템을 직접 크롤링하여 본문 검색을 수행하는 On-the-fly 엔진.
*   **적용**: 별도의 데이터베이스 구축이 부담스러운 MVP 프로젝트에 적합합니다.

---

## 4. 지식 자산화 (AI 협업 가이드)

향후 다른 프로젝트에서 이 프로젝트의 코드를 AI에게 참조시키려면 다음의 프롬프트를 활용하세요.

> **"Onrivi Author 프로젝트의 [파일 트리 정렬 로직 / 보안 체크 로직]을 참조해서, 현재 프로젝트의 파일 관리 모듈을 작성해줘."**

*   **참고**: 이 프로젝트의 `docs/` 폴더 전체를 새 프로젝트의 `reference/` 폴더에 넣어두면, AI가 기존 설계 의도를 완벽히 파악하여 일관성 있는 코드를 생성할 수 있습니다.
